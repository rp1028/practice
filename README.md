# 요구사항 명세서

**프로젝트명:** 전기차 전환 시대, 지역별 전기차 이용 환경 분석 및 정보 제공 시스템
**작성일:** 2026.03.11
**최종 수정일:** 2026.03.18

---

## 1. 기능 요구사항

| 구분 | 요구사항명 | 상세 설명 | 비고 |
|:---:|:---|:---|:---|
| 기능 | 지역별 전기차 등록 현황 조회 | 17개 시도별 전기차 누적 등록대수를 월별·연도별로 조회할 수 있어야 한다 | 2010~2026년 데이터 |
| 기능 | 전기차 등록 추이 시각화 | 지역별 등록 추이를 선 그래프, 막대 그래프, 히트맵 등으로 시각화해야 한다 | |
| 기능 | 전년 대비 증감 분석 | 전년 동월 대비 증가량 및 점유율 변화를 계산하여 제공해야 한다 | |
| 기능 | 충전소 위치 검색 | 사용자가 지역명을 입력하면 해당 지역의 전기차 충전소 목록을 반환해야 한다 | 카카오맵 API |
| 기능 | 충전소 지도 시각화 | 검색된 충전소 위치를 지도 위에 마커로 표시하고, 클릭 시 상세 정보를 팝업으로 보여야 한다 | |
| 기능 | 충전소 운영 정보 제공 | 충전기 유형(급속/완속), 운영 상태, 이용 가능 여부를 필터링하여 조회할 수 있어야 한다 | |
| 기능 | 충전 요금 조회 | 운영사별(환경부, 한국전력, E-pit 등) 충전기 유형별 단가 기준표를 제공해야 한다 | |
| 기능 | 충전 비용 계산기 | 충전량(kWh)과 요금 유형을 선택하면 예상 충전 비용을 산출해야 한다 | |
| 기능 | 보조금 정책 조회 | 2026년 국고 보조금 기준표를 차량 유형(승용·화물·승합·이륜 등)별로 제공해야 한다 | |
| 기능 | 보조금 필터링 | 제조사 및 차종으로 필터링하여 해당 보조금 금액을 조회할 수 있어야 한다 | |
| 기능 | 브랜드별 FAQ 제공 | 테슬라, 현대, 기아 등 제조사별 전기차 관련 자주 묻는 질문과 답변을 제공해야 한다 | 웹 크롤링 기반 |
| 기능 | 서울시 전기차 FAQ 제공 | 서울시 전기차 충전 인프라 포털의 FAQ를 카테고리별로 분류하여 제공해야 한다 | 웹 크롤링 기반 |
| 기능 | FAQ 카테고리 필터 | 차량군(전기승용·화물·승합, 전기이륜 등)으로 FAQ를 필터링하여 조회할 수 있어야 한다 | |
| 기능 | 뉴스 키워드 분석 | "전기차 보조금" 관련 뉴스 기사에서 자주 등장하는 단어를 집계하여 차트로 제공해야 한다 | 네이버 뉴스 크롤링 |
| 기능 | 워드클라우드 시각화 | 뉴스 키워드 분석 결과를 자주 나온 단어일수록 크게 표시하는 이미지로 시각화해야 한다 | |

## 2. 비기능 요구사항

| 구분 | 요구사항명 | 상세 설명 | 비고 |
|:---:|:---|:---|:---|
| 성능 | 데이터 캐싱 | 같은 데이터를 반복 불러올 때 불필요한 재처리 없이 빠르게 표시되어야 한다 | |
| 가용성 | DB 없이 동작 | MySQL 연결이 없어도 CSV 파일 기반으로 대시보드가 정상 동작해야 한다 | |
| 보안 | API 키 보호 | 카카오, 네이버 API 키 및 DB 접속 정보는 별도 파일로 관리하고 외부에 노출되지 않아야 한다 | .env 파일, .gitignore |
| 유지보수성 | 데이터·UI 분리 | 데이터 처리 로직과 화면 렌더링 로직을 분리하여 모듈별 독립 수정이 가능해야 한다 | |
| 데이터 품질 | 지역명 표준화 | 여러 데이터 소스의 지역명을 17개 표준 시도명으로 통일하여 일관성을 유지해야 한다 | |
| UI/UX | 사이드바 네비게이션 | 사이드바를 통해 7개 메뉴 간 이동이 가능해야 한다 | |
| UI/UX | 필터 및 검색 | 주요 화면에서 지역, 기간, 차종 등 조건 필터를 제공해야 한다 | |
